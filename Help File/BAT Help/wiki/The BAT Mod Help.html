1. What is The BAT Mod?

What is The BAT Mod?

BtS Artistic Tweaks Mod (BAT Mod) is a modification for Civilization IV: Beyond the Sword. Civlization 4 was designed to be extremely modable, but not all modifications are compatible with each other. In an effort to combine these many different mods from the Civilization Community into one concise program, The BUG Project was born. The BAT Mod is an offshot of the original BUG Mod. It includes new graphics to make each civilization as unique as they are in reality.

BAT combines mods that do not change the basic game play of Civilization 4 while enhancing the interface and improving ease of use. Some aspects that we feel are within these criteria are Advisor Screens, the City Screen, the Main View, the Civilopedia Screens, etc. Also, any additions or changes made will not provide any information that cannot be figured out within the game itself.

We feel this Mod is safe for use in any Succession Game, but all players will have to run it from the Mods folder. If you find any changes within the code that modify gameplay, please contact us, and we will make it our priority to fix the problem.

2. How to Install

Installation

 

Self-Extracting .EXE instructions:

1) Run "BATMod_xxx.exe" file
2) If the directory listed is not your BtS Mods folder, then please, browse for your CIV:BtS Mods folder.
3) Restart Civ4.



ZIP File instructions:

1) Locate your CIV:BtS Mods folder (Usually within "My Documents\My Games\Beyond the Sword\Mods").
2) Copy contents of "BATMod_xxx.zip" into your CIV:BtS Mods folder.
3) Restart Civ4.

3. Changes made to original Mods

Changes made to original Mods

Below are changes that we have made to the original code. The majority of changes made were required for BtS play. Nothing we have done will alter the rules or your gameplay in any way.

 

Great Person Progress Bar

  • Optionally displays the percent chances of each GP type.
  • Clicking the text in the bar zooms to the city.


Autolog (enhanced by Ruff_Hi prior to merging with BUG)

  • Silent logging mode
  • Default file name
  • User controlled logging start / stop
  • Logging of pillaging
  • Logging of Whipping
  • Logging of War status
  • Logging of Civic changes
  • Logging of Religion changes
  • Location and probability of combat
  • While Attacking / Defending status added to combat
  • IBT added to split user actions and AI actions
  • Option to treat 4000BC as turn 0
  • Date and Time of turn
  • Expected duration on builds and tech research
  • Color coded without '"' for forums that don't support them

 

Civ4lerts

  • Rewrote the majority of code from scratch to fix the timing bugs with most alerts.
  • Added alerts for when a civ is willing to sign Open Borders, Defensive Pact and Permanent Alliance agreements.
  • Added alerts for when production in a city can be hurried (by population or gold).
  • Extended pending growth alert to include actual growth and pending/actual shrinkage of a city.
  • Changed happy/healthy alerts to notify whenever the city changes state either way.
  • Created new options for alerts that shared options with other alerts.


Reminders

  • Events are saved and loaded.
  • When a reminder pops up, you have the option to show the reminder again the next turn.
  • You can choose how reminders are displayed: in the event log, as a modal pop up or both.
  • Reminders can be logged using Autolog.
  • Reminders that have fired at the start of the turn are shown as the flashing text at the end of your turn.

 

Better Espionage Screen

  • Rearranged fields for clarity.
  • Added tri-color coding to espionage cost modifier percentages: default, good and bad.
  • Added tri-color coding to missions (both passive and active): cannot perform, can perform and can almost perform.



Customizable Domestic Advisor

  • Added BtS columns (e.g. corporations and espionage) and "Liberate Colony" button.
  • Added other new columns (e.g. turns until culture growth and great person birth, and “hurry pop/gold” columns).
  • Added a set of building class columns that show whether or not you have constructed the building type (based on your current civilization) for that class (instead of showing the building’s effect). The headers for these columns are the building icons, and the hover text shows the building’s effect.
  • Resource columns now show their icon as the header and the effects or the availability as the value (2 different series).
  • Fixed crash caused when you had more than 20 or so cities.
  • Added page switching buttons.
  • Added buttons to move pages while customizing.
  • Added a button to toggle whether or not to display the controls for changing specialists for a page.



Raw Commerce

  • Added the 50% bonus for the capital under the Bureaucracy civic.
  • Added a button to toggle between commerce and production.

6.2 Autlog

Autlog

Autologger by eotinb

 

About Autologger

A mod to facilitate note taking, with succession games in mind.

 

Instructions

The autologger can be configured through the BUG Options screen. The Default directory where the logs are stores is in the My Docs\My Games\Beyond the Sword\Autolog directory. The default file name is the name you used as your Leader name. You can change these from within the BUG Options screen. The shortcut commands you should know for the autologger are:

  • Alt + L - Toggle autoLogging

  • Alt + B - Create Battle dump

  • Alt + E - Add custom Entry to autologger.

You can also change the format used. The options are HTML, Plain Text, Forum Codes, and Forum Codes with color. Default is Forum Codes, which is designed to be copy and pasted straight to a forum such as CFC, and posted to your story or succession games.

One word of warning. Using of autologger can be spammy, and take much of the heart of writing up a succession game report. A suggested method of using it is to write a summary of what happened, with many screen shots, and embedding the Autolog dump into a Spoilers tab within your message. Refer to the documentation on the forum you use on how to do this.

6.9 Civ4lerts

Civ4lerts

Civ4lerts by Dr. Elmer Jiggles (Heavily Modified by EmperorFool)

 

About Civ4lerts

This mod implements on screen alerts for a variety of in-game events such as city growth and the availability of new trade opportunities. The concept is modeled after the CivAssist tool for Civilization 3, and in fact, many of the alerts are the same.

The alerts will not reveal information that would otherwise be unavailable. They are simply intended to call the player's attention to situations as they arise without requiring tedious scanning of the advisor screens on each turn.

The alert messages are integrated with the existing event log. Like those messages, a notification is displayed on screen for a short period of time. Messages that have been cleared from the screen can be recalled by opening the event log. When appropriate, clicking on a message brings the camera focus to the site of the event.

The Alerts you can set up are:

Cities:

  • Pending Growth - Alert displayed the turn before city growth.

  • Growth - Alert displayed when a city grows.

  • Pending Healthiness - Alert displayed the turn before city becomes unhealthy.

  • Healthiness - Alert displayed when a city grows unhealthy.

  • Pending Happiness - Alert displayed the turn before city becomes unhappy.

  • Happiness - Alert displayed when a city grows unhappy.

  • Pending Border Expansion - Alert displayed the turn before a cities cultural borders will expand.

  • Can Hurry w. Population - Can use population to hurry the production of a city, as well as how many population points it will take, and the amount of hammers of overflow.

  • Can Hurry W. Gold - Can hurry the production of a city with cash, and how much it will cost.


Diplomacy:

  • Open Border - Will sign Open Borders Agreement.

  • Defensive Pact - Will sign Defensive Pact

  • Permanent Alliance - Will sign Permanent Alliance

 

Trading:

  • Technologies - What technologies other civilizations will trade.

  • Gold - When other civilizations have Gold available for trade. You can set the minimum amount of gold before the alert is triggers as well.

  • Gold Per Turn - When other civilizations have Gold Per Turn available for trade. You can set the minimum amount of gold before the alert is triggers as well.

 

Victory:

  • Domination Limit: Population - When you are about to win a Domination Victory for Population. You can set the percentage away from that limit before the alert is triggered.

  • Domination Limit: Land - When you are about to win a Domination Victory for Land. You can set the percentage away from that limit before the alert is triggered.

Instructions

From the BUG Options Screen, in the Alerts Tab, you can check the alerts you want, or don't want, to be shown at the start of every turn.

Unit Naming

Unit Naming by ruff_hi

  1. Overview

This document explains the unit naming convention used in the BUG Mod. It documents what is meant to happen, not necessarily what actually happens. If you find a situation where this document differs from the unit name actually generated within the game, then please report it as a bug.

  1. Unit Naming Convention

The following is the code that controls the name that your unit will have.

  • ^civ4^ - no naming convention, uses standard civ4

  • ^rd^ - random name

  • ^rc^ - random civ related name

  • ^ct^ - City

  • ^cv^ - Civilization

  • ^ut^ - unit (eg Archer)

  • ^cb^ - combat type (Melee)

  • ^dm^ - domain (Water)

  • ^ld^ - leader

  • ^cnt[f]^ - count across all units (increments based on unit)

  • ^cntu[f]^ - count across same unit (increments based on unit)

  • ^cntct[f]^ - count across same city (increments based on unit)

  • ^cntuct[f]^ - count across same unit / city (increments based on unit)

  • ^cntc[f]^ - count across same combat type (increments based on combat type)

  • ^cntd[f]^ - count across same domain (increments based on domain)

  • ^tt1[f][x:y]^ - total where the total is a random number between x and y (number)

  • ^tt2[f][x]^ - total count (starts at x, incremented by 1 each time ^tt is reset to 1)



There are a number of number formats, as follows:

  • [f] can be either 's', 'A', 'a', 'p', 'g', 'n', 'o' or 'r' for ...

    • s - silent (not shown)

    • A - upper case alpha (A, B, C, D, ...)

    • a - lower case alpha (a, b, c, d, ...)

    • p - phonetic (alpha, bravo, Charlie, delta, echo, ...)

    • g - greek (alpha, beta, gamma, delta, epsilon, ...)

    • n - number

    • o - ordinal (1st, 2nd, 3rd, 4th, ...)

    • r - roman (I, IV, V, X, ...)

Some number formats have a built in max (i.e. alpha). In these situations, the numbering loops back to the start. For example, if you pass the number ‘30’ and ask for the number to be formatted with an ‘A’ (upper case alpha) then it will return ‘D’.

About Whip Assistant

Whip Assistant will tell you how many hammers of overflow you will have if you hurry production by sacrificing your population. The overflow you receive is put towards your next build. So, for example, you can whip a library for 2 population with 32 hammer overflow means that it will take 2 of your population, but you will get the library the next turn, plus 32 hammers to whatever you build next.

Civilization IV: Beyond the Sword

Il BtS Unaltered Gameplay Mod (BUG Mod) è un mod per il gioco Civilization IV: Beyond the Sword.
Civ4 è un gioco facilmente moddabile, e la comunità di Civ4 ha realizzato un gran numero di mod, ma spesso i diversi mod non sono compatibili l’uno con l’altro.
Il progetto BUG Mod è nato proprio dallo sforzo di combinare i diversi mod prodotti dalla comunità di Civ4.

 

Il BUG Mod si prefigge di combinare in un unico mod omnicomprensivo tutti i mod che rientrano nella categoria degli “Unaltered Gameplay Mods”, ossia quei mod che lasciano inalterato il gameplay del gioco originale, ma rendendo il gioco più facile e piacevole (dal punto di vista grafico e del controllo).
In particolare, rientrano in questa categoria i mod che agiscono sulla schermata principale di gioco, sulla schermata cittadina, sugli schermi dei vari consiglieri, sulla Civipedia, eccetera. Inoltre, le aggiunte ed i cambi fatti alle varie interfacce non devono fornire informazioni che non siano comunque reperibili altrimenti nel gioco.

 

Riteniamo che questo mod possa essere utilizzato senza problemi in ogni Succession Game, anche in quelli che utilizzano un altro mod. Se trovate qualche modifica al gameplay nel nostro mod, per favore contattateci e fatecelo sapere, vi garantiamo che faremo della risoluzione di tale problema una nostra priorità.

 

Se volete che il vostro mod venga aggiunto al BUG Mod, o volete contattarci per qualche suggerimento, visitate il sito di sviluppo del mod su SourceForge:

INSTALLAZIONE

Attenzione!
L’installazione mediante file .exe sovrascriverà la vostra cartella “Beyond the Sword\CustomAssets”, cancellando tutte le modifiche ivi contenute. Pertanto vi consigliamo di fare un backup preventivo della vostra cartella “Beyond the Sword\CustomAssets”.
Se volete mantenere operative le modifiche che avevate nella vostra cartella “CustomAssets”, avete due possibilità:
1) Unire le due cartelle “CustomAssets” (la vostra e la nostra) in una unica, avendo cura di editare i vari file che dovessero essere presenti in entrambe in modo da fonderli in uno unico. Questa procedura può essere lunga e difficoltosa (a seconda del numero di file presenti nella vostra cartella “CustomAssets” che sono presenti anche nella nostra e delle vostre competenze nell’editing dei file .py e .xml), per cui ci teniamo ad avvertirvi di non avventurarvici a meno che non sappiate cosa state facendo: se la fusione non viene eseguita correttamente, probabilmente non funzioneranno né il BUG mod né le vostre modifiche antecedenti la sua installazione.
2) Installare il BUG Mod nella cartella Beyond the Sword\Mods\BUG Mod (invece che nella directory principale Beyond the Sword) e successivamente rinominare la cartella “CustomAssets” in “Assets”. In questo modo, dovrete caricare il BUG Mod ogni volta che lanciate il gioco. Inoltre, in questo modo, tutti gli altri giocatori che vogliono giocare con voi in Multiplayer o in un Succession Game dovranno avere installata la vostra stessa versione del BUG Mod.
Se utilizzate questo secondo metodo e non volete caricare il mod manualmente ogni volta che lanciate il gioco, potete modificare il parametro “Mods” nel file “CivilizationIV.ini” da “0” a “BUG Mod”. Questro dirà a Civ4 di caricare il BUG Mod automaticamente. Per utilizzare un altro mod, o (nessun mod), basta selezionare “UNLOAD MOD” nel menù principale.

Installazione da file .exe
1) Lanciate il file BUGMod_20.exe.
2) Quando vi viene chiesta la directory di installazione, cercate la cartella di Civ4:BtS all’interno della cartella Documenti (solitamente Documenti\My Games\Beyond the Sword).

Installazione da file .zip
1) Individuate la cartella di Civ4:BtS all’interno della cartella Documenti (solitamente Documenti\My Games\Beyond the Sword).
2) Dezippate il file BUGMod_20.zip all’interno della cartella Beyond the Sword, avendo cura di conservare i path impostati nel file zip: in particolare, il file BUG.ini deve finire direttamente all’interno della cartella Beyond the Sword, mentre gli altri file devono finire all’interno delle relative sub-directory della cartella Beyond the Sword\CustomAssets.

 Advanced Scoreboard v. 1.0 (by EmperorFool)


Compatibile con Beyond the Sword 3.0.x


DESCRIZIONE


Questo mod fornisce un pannello dei punteggi completamente personalizzabile come modulo del BUG Mod

<http://forums.civfanatics.com/forumdisplay.php?f=268>.

Integra sia il mod Dead Civ Scoreboard di The Lopez sia il mod Attitude Icons di Porge (entrambi sono opzionali).

Consente di specificare esattamente quali parti mostrare ed in che ordine. Inoltre aggiunge la possibilità di mostrare le icone invece dei nomi delle tecnologie che gli avversari stanno ricercando (ovviamente quando siete abilitati a vederle).


ISTRUZIONI


Potete personalizzare questo mod utilizzando la Schermata delle Opzioni del BUG Mod (Ctrl-Alt-O), nella tabella “Scoreboard”. Le opzioni sono nella colonna all’estrema destra, sotto “Advanced Layout”. Selezionando la casella “Enabled”, potrete personalizzare la disposizione delle voci nel pannello del punteggio.


* La casella di testo “Display Order” consente di scegliere che oggetti mostrare ed in che ordine. Ad ogni oggetto corrisponde un codice letterale. Alla casella di testo è associato un suggerimento (che compare lasciandoci sopra per un paio di secondi il cursore del mouse) che ricorda i vari codici letterali ed il loro significato, che è il seguente:

S il punteggio della Civiltà

C il nome della Civiltà e/o del Leader (in base a quanto impostato nella casella di scelta opzionale “Display Name

R la religione di stato della Civiltà

? mostra un punto interrogativo se non avete ancora incontrato la Civiltà (probabilmente serve per i membri ancora non incontrati di squadre già contattate)

W mostra l’icona di un pugno se siete in Guerra con la Civiltà

E mostra l’icona della spia se avete un rapporto di punti spionaggio positive nei confronti della Civiltà

P il vostro rapporto di forza con la Civiltà

T la tecnologia che la Civiltà sta ricercando (nota per via del suo vassallaggio nei vostri confronti, della sua alleanza con voi o del vostro spionaggio)

U il numero di turni di ricerca rimasti (vedi T)

N mostra l’icona del triangolo di frecce quando la civiltà è connessa alla vostra rete commerciale

B mostra l’icona di una pergamena quando avete firmato un accordo di libero accesso con la Civiltà

D mostra l’icona di una pergamena (diversa dalla precedente) quando avete firmato un accordo difensivo con la Civiltà

A l’atteggiamento della Civiltà nei vostri confronti

* siete in attesa che questa Civiltà complete il suo turno

L il valore di ping del giocatore che guida la Civiltà

O il giocatore che guida la Civiltà è “out-of-sync”

Gli ultimi 3 sono visibili solo per gli avversari in una partita in multiplayer, i primi 3 sono visibili anche per la vostra civiltà, gli altri 10 sono visibili solo per gli avversari.


* La casella di spunta “Left-Align Names” consente di allineare a sinistra anziché a destra i nomi delle civiltà e/o dei leader.


* La casella di spunta “Technology Icons” mostra le tecnologie ricercate dagli avversari sotto forma di icone invece che nomi.


NOTA PER I MODDER


Se desiderate utilizzare questo mod all’interno del vostro mod, dovrete necessariamente fare estese modifiche python al file CvMainInterface.py e copiare il file Scoreboard.py dalla cartella Python/BUG. La spiegazione dettagliata dei cambiamenti mi richiederebbe più tempo di quello impiegato per farli, perciò se qualcuno realmente desidera includere questo mod nel suo, mi contatti direttamente (PM EmperorFool) e cercheremo di sistemare la cosa insieme. Ovviamente siete i benvenuti se volete provarci da soli. Probabilmente è più facile partire con un file CvMainInterface.py standard a cui aggiungere il necessario. Tutti i cambiamenti sono dove c’è il codice che genera il pannello del punteggio.


Se includete questo mod in un vostro mod, tutto quello che vi chiedo è di fare riferimento al BUG Mod da qualche parte nel vostro mod o nella sua documentazione: <http://forums.civfanatics.com/forumdisplay.php?f=268>

 Autolog v. 1.1 (by eotinb)


Un mod per agevolare chi vuole prendere delle note, realizzato pensando soprattutto ai Succession Games.


CAMBIAMENTI NELLA VERISONE 1.1


Nuovi eventi registrati:

città perse ed acquisite tramite combattimenti e scambi;

primi contatti con un’altra civiltà.


Ulteriori informazioni aggiunte agli eventi già registrati in precedenza:

le note di combattimento ora mostrano la forza del vincitore dopo la battaglia;

ora viene registrato anche quando è un’altra civiltà a scoprire una religione (precedentemente veniva registrato solo quando la scopriva il giocatore) e anche quando una religione di cui controllate la Città Santa si diffonde (precedentemente, la diffusione di una religione veniva registrata solo quando avveniva in una città sotto il vostro controllo).


Nuove caratteristiche:

E’ stato eliminato il fastidioso “problema dell’auto-salvataggio (“autosave quirk”): se non sapete di cosa si tratta, non preoccupatevene, e siate felici di non averci avuto a che fare.

Le note create con Alt+E vengono ora mostrate sia nel file di testo che nel registro on-line e sono precedute da una notifica che sono state create dal giocatore (la notifica può essere modificata mediante la variabile “prefixTag” nel file autolog.py).

Ora si possono visualizzare i cambiamenti nel file del registro mentre il gioco è in corso (anche se non è possibile editarlo finché il gioco è attivo, a meno di non attivare un nuovo file di registro).

Ora si può scegliere il nome del file di registro all’inizio di una nuova partita o quando la si carica, altrimenti verrà utilizzato il nome di default (autolog.txt).

Ora c’è una opzione per ottenere le note colorate in modo adatto per i messaggi sul forum (si veda la variabile “color” nel file autolog.py).


Caratteristiche rimosse:


Tutte le aggiunte al registro in-game degli eventi sono state rimosse. Presto realizzerò un altro mod che rimpiazzerà alcune di queste funzionalità e ne aggiungerà altre. Inoltre, è stata disabilitata la short cut Ctrl+E.


IMPOSTAZIONI


Potete selezionare il nome di default per il file di registro esterno. Per prima cosa, controllate la cartella in cui volete che vada a finire il file (LA CARTELLA DI DEFAULT PUO’ NON FUNZIONARE SUL VOSTRO PC, perciò controllatela): aprite il file autolog.py con un editor di testo e cercate la stringa “path =” ed assicuratevi che dopo il segno “=” ci sia un percorso di cartella valido. La maggior parte dei problemi con questo mod sono dovuti a problemi di percorso, per cui controllate che le cartelle elencate nel percorso siano separate da “\\” e che l’ultima di esse NON sia seguita da un “\”; ad esempio, "C:\\games\\civ4" è un percorso corretto, mentre non lo sono "C:\games\civ4" e "C:\\games\\civ4\".


Se non modificate il nome del file o rinominate il registro precedente, il nuovo registro verrà aggiunto in calce al precedente avente lo stesso nome; questo avviene in modo da evitarvi di sovrascrivere accidentalmente i vostri registri precedenti.


Potete scegliere che tipo di tags (<HTML>, [forum], nessuno) utilizzare per la formattazione utilizzando la variabile "style" in autolog.py. Inoltre, potete scegliere se le note formattate per il forum debbano essere codificate cromaticamente, utilizzando la variabile “color” nel file autolog.py. E’ anche possibile scegliere il colore per ogni evento, ma non è immediato, è necessario entrare in profondità nel codice (se volete farlo e vi serve qualche aiuto da parte mia, chiedete pure liberamente nel thread relativo a questo mod su CFC).


CARATTERISTICHE


La combinazione di tasti Alt+E consente di introdurre una nota scritta dal giocatore nel registro (sia nel file testuale esterno che nel registro in-game e come nota sulla schermo). Le note così introdotte sono precedute da una notifica che sono state create dal giocatore (la notifica può essere modificata mediante la variabile “prefixTag” nel file autolog.py).


Gli eventi registrati nel file di registro sono:

La fondazione di una città, la crescita di una città, l’espansione dei confine di una città, l’acquisto o la perdita di una città, la razzia di una città, l’inizio e la conclusione dell’apprendimento di una tecnologia, l’inizio e la conclusione della costruzione di un edificio o una unità, gli esiti di un combattimento, l’inizio e la fine di una età dell’Oro, la nascita di un Grande Personaggio, l’esito dell’esplorazione di un villaggio tribale, la promozione di un’unità, la scoperta e la diffusione di una religione.


Tutte le tipologie di evento registrate possono essere attivate o disattivate individualmente modificando opportunamente il file autolog.py, in cui possono anche essere trovati maggiori dettagli su ogni tipologia di evento.


PROBLEMI


Il risultato dell’esplorazione di un villaggio tribale è gestito dal gioco come un numero intero; ho individuato alcuni di essi mediante una serie di prove ed errori, ma alcuni risultati sono piuttosto rari ed in ogni caso non li ho individuati ancora tutti. Per ora, se il risultato è qualcosa per cui non ho ancora identificato il codice, viene riportato il codice stesso; se per voi è importante registrare il tipo di evento in questo caso, utilizzate una nota personale (Alt+E). Per favore, comunicatemi ogni ulteriore codice che individuate (o eventuali correzioni a quelli che ho individuato), in modo da aiutarmi a fare i cambiamenti e le aggiunte necessari.


CARATTERISTICHE DA AGGIUNGERE


La possibilità di modificare in-game quello che deve essere registrato (invece che modificarlo attraverso le modifiche al file autolog.py).

La possibilità di esportare il registro attualmente attivo in-game.

Una qualche forma di coordinamento con gli screenshot.

DESCRIZIONE


Questa è una tabella di colori addizionale per il gioco Sid Meier's Civilization 4. Non è un mod a sé stante, ma una risorsa XML che può essere utilizzata dalla comunità dei modder di Civ4.

Il gioco originale di Civ4 è uscito con una gamma limitata di valori per i colori. Ciò limita i modder ed i giocatori nella scelta di nuovi schemi di colore. L’aggiunta di ulteriori colori è possibile, ma richiede un lavoro di editing XML noioso e suscettibile di errori; inoltre, l’utilizzo di diverse tabelle di colori è una fonte di incompatibilità per i mod.

Ciò smette di essere un problema utilizzando questa tabella dei colori.

Questo file ha una ampia gamma di colori (più di 150), in modo che difficilmente sia necessaria una aggiunta ulteriore di colori. E’ possibile che ci sia qualche doppione (lo stesso colore con nomi diversi) ma questo non causa alcun problema.


COLORI AGGIUNTIVI


1. colori base HTML (16 colori);

2. colore arancio, con trasparenza al 25%, 50% e 75% (4 colori);

3. colori coi nomi provenienti dal file rgb.txt X11 Window system (140 colori);

4. livelli di grigio dallo 0% (nero) al 100% (bianco) con aumenti progressivi del 10% (11 colori).

Le prime due categorie di colori cominciano con il prefisso "COLOR_W_..." (ad esempio: COLOR_W_AQUA, COLOR_W_BLUE, COLOR_W_ORANGE, COLOR_W_THIN_CLEAR; si noti che il colore COLOR_W_GREEN è diverso da COLOR_GREEN (che corrisponde invece a COLOR_W_LIME)); le ultime due categorie di colori cominciano con il prefisso "COLOR_X_..." (ad esempio: COLOR_X_ALICEBLUE, COLOR_X_BLUEVIOLET, COLOR_X_GOLD, COLOR_X_GRAY80; si noti che il colore COLOR_X_ORANGE è leggermente diverso dal colore COLOR_W_ORANGE).

 Sevopedia ver 2.3.1 (by Sevo)


Compatibilità verificata con Beyond the Sword 3.02


Io amo Civ4, ma tutti noi sappiamo che la Civilopedia è un punto debole. E’ impossibile da consultare facilmente, e la sua struttura è anti-intuitiva.

Perciò, questo è un miglioramento: Sevo's Civilopedia (o Sevo-lo-pedia, o anche Sevopedia, ma sto divagando). Questo mod trasforma la Civilopedia in una fonte di riferimento più bella, user-friendly e facile da consultare.


CARATTERISTICHE


* Menù spostato a sinistra, con il menù principale sull’estrema sinistra ed il sottomenù alla sua destra.

* Menù principale riordinato in categorie, caratterizzate da icone

* Sottomenù organizzati in un elenco facile da leggere.

* Niente più salti fuori dal menù principale quando si seleziona una pagina: la pagina selezionata si apre nello spazio disponibile sulla destra.

* Ripristino delle voci precedentemente nascoste (per esempio il leader barbaro)

* Facilità di integrazione con quasi tutti i mod esistenti

* Possibilità opzionale di disabilitare le animazioni


* Vovan's Unit Upgrade Chart: genera automaticamente uno schema dei miglioramenti per le unità (anche per le versioni modificate dei mod!)

* Progor's Unit Promotion Chart: genera automaticamente uno schema delle promozioni per le unità (anche per le versioni modificate dei mod!)

* Fitchn's Civilopedia Index: un semplice elenco alfabetico di tutto il contenuto della Civilopedia, funzionante in tutte le lingue.


CONFIGURAZIONE


 PEDIA_SHOW_ANIMATIONS:

Esaudendo le richieste, ho incluso l’opzione per disattivare le animazioni nelle schermate della Civilopedia. Disabilita le animazioni per le unità, gli edifici, le risorse ed i miglioramenti, mentre le animazioni per i leader rimangono attive. L’impostazione di default lascia le animazioni attive.

 PEDIA_SHOW_CIV_NAME_FOR_UNIQUE:

Mostra i nomi delle civiltà per le unità e gli edifici unici negli elenchi secondari. Siccome ciò può non essere appropriato per alcuni mod, può essere disabilitato. L’opzione di default è attiva.

 PEDIA_SHOW_GRAPHICAL_ONLY:

Se abilitata, Barbarian Leader, Minor Civ, Lead by Warlord, ecc. sono nascosti nella Sevopedia. L’opzione (richiesta da Naze) di default è disabilitata.

 PEDIA_SKIP_BLANK_PAGE:

Se abilitata, la prima sottocategoria verrà mostrata automaticamente quando si clicca sul link di una categoria principale. L’opzione (richiesta da Impaler[WrG) di default è disabilitata.

 PEDIA_SKIP_BLANK_PAGE:

Questa opzione nasconde nella Sevopedia la cui “text key” sia la stessa dell’unità di default per la sua classe; opzione molto utile per i mod come Ethnically Diverse Units. L’opzione di default è disabilitata.


<bGraphicalOnly> (XML tag):

E’ possibile aggiungere alle voci il tag XML <bGraphicalOnly>; tali voci verranno filtrate dalla Sevopedia se l’opzione PEDIA_SHOW_GRAPHICAL_ONLY è attiva (ossia 1). Ho fornito lo schema d’esempio solo per Civ4UnitSchema.xml. Potete scrivervi da soli gli schemi per ogni altro tipo di informazione.


Opzioni per filtri personalizzati:

Da CvPediaScreen.isFiltered(), dovrebbe essere abbastanza facile utilizzare le opzioni dei filtri “graphical only” e “hide identical” come modelli per scrivere le vostre opzioni per filtri personalizzati. Ciò potrebbe esservi utile se voleste, per esempio, nascondere tutti gli animali. Per essere portata a termine, questa operazione richiede alcune conoscenze basilari del Python.

Denominazione delle Unità nel BUG Mod



Questo documento spiega la convenzione per la denominazione delle unità utilizzata dal BUG Mod.

Se trovate un caso in cui questo documento differisce da quello che avviene realmente nel gioco, per piacere riferitecelo.


Quello di seguito è il codice utilizzato per generare il nome delle vostre unità.


^civ4^ nessuna denominazione, utilizza I nomi standard di civ4

^rd^ nome random

^rc^ nome random relativo alla Civiltà

^ut^ tipo di unità (ad esempio “arciere”)

^cb^ tipo di combattimento (ad esempio “corpo a corpo”)

^ld^ leader

^ct^ Città

^cv^ Civiltà

^dm^ ambiente (ad esempio “acqua”)

^cnt[f]^ conteggia su tutte le unità

^cntu[f]^ conteggia su tutte le unità dello stesso tipo

^cntct[f]^ conteggia su tutte le unità della stessa città

^cntuct[f]^ conteggia su tutte le unità dello stesso tipo e città

^cntc[f]^ conteggia su tutte le unità dello stesso tipo di combattimento

^cntd[f]^ conteggia su tutte le unità dello stesso ambiente

^tt1[f][x:y]^ totale del conteggio, dove il totale è un numero random fra x e y (vale x se y=x)

^tt2[f][x]^ totale del conteggio (parte da x, incrementato di 1 ogni volta che tt1 torna a 1).


I diversi formati numerici sono i seguenti: [f] può essere:

s silente (non mostrato)

A maiuscole (A, B, C, D…)

a minuscole (a, b, c, d…)

p fonetiche (alpha, bravo, charlie, delta…)

g greche (alpha, beta, gamma, delta…)

n numeri (1, 2, 3, 4…)

o ordinali (1st, 2nd, 3rd, 4th…)

r romani (I, II, III, IV…)

Alcuni formati prevedono un massimo (A, a, p, g): quando viene raggiunto, la numerazione riprende dall’inizio. Per esempio, se scegliete il formato A, la trentesima unità sarà indicata con “D”.


Più nel dettaglio:

^civ4^

Questa indicazione prevale su tutte le altre. Per l’unità in questione verrà utilizzato il nome standard di Civ4, indipendentemente dalle altre indicazioni. Per esempio, la stringa ^civ4^ di ^ct^ fornirà come risultato il nome standard dell’unità, e la parte “di ^ct^” verrà ignorata.

^rd^

Questa indicazione prevale su tutte le altre, a parte ^civ4^. Per l’unità in questione verrà utilizzato un nome random. Per esempio, la stringa Armata ^cv^ ^rd^ fornirà come risultato un nome casuale, e la parte “Armata ^cv^” verrà ignorata.

^rc^

Questa indicazione prevale su tutte le altre, a parte ^civ4^ e ^rd^. All’unità verrà assegnato un nome casuale relativo alla civiltà cui appartiene. Valgono considerazioni analoghe al caso precedente.

^ut^

Questa indicazione assegna all’unità come nome quello del tipo di unità (ad esempio “Arciere”).

^cb^

Questa indicazione assegna all’unità come nome quello del tipo di combattimento dell’unità.


Vediamo ora alcuni esempi.


  • ^ut^ (^cv^), se applicata ad un arciere romano, fornirà come risultato “Arciere (Roma)”.

  • ^ut^ cnt[r], se applicata alle prime due unità prodotte, un arciere ed un guerriero, fornirà come risultato “Arciere I” e “Guerriero II”. In questo caso, il conteggio prosegue all’infinito senza reset.

  • Armata ^cnt[A]^ ^tt1[s][2:2]^, se applicata alle prime tre unità prodotte, un arciere, un guerriero ed uno scout, fornirà come risultato “Armata A”, “Armata B” e “Armata A”. In questo caso, il conteggio non prosegue all’infinito, ma si resetta quando viene raggiunto il massimo indicato da tt1, che in questo caso non è un numero random ma vale 2 (perché x=y=2) e non viene mostrato (a causa del parametro di formato numerico s).

  • ^cntc[n]^ di ^tt1[n][6:10]^, Unimatrice ^tt2[n][1]^, formulazione più complessa (non per niente è quella utilizzata dai Borg ^_-), che conteggia le unità dello stesso tipo di combattimento, fino ad un totale (determinato da tt1) dato da un numero casuale compreso fra 6 e 10; ogni volta che il contatore si resetterà per aver raggiunto tale massimo, il numero indicato da tt2 salirà di una unità (partendo da 1). Il risultato fornitò sarà qualcosa del tipo: …, “6 di 7, Unimatrice 1”, “7 di 7, Unimatrice 1”, “1 di 9, Unimatrice 2”, …
